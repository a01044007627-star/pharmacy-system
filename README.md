# Pharmacy System

منظومة إدارة صيدليات متعددة الفروع مبنية بـ Next.js وTypeScript وSupabase، وتدعم التشغيل Online/Offline والصلاحيات والمبيعات والمشتريات والمخزون والحسابات.

## بيئة التشغيل

- Node.js 20.x
- pnpm 10.12.x
- Supabase
- Vercel

```bash
corepack enable
pnpm install --frozen-lockfile
pnpm dev
```

## أوامر الجودة

```bash
pnpm typecheck
pnpm test
pnpm lint:ci
pnpm audit:project
pnpm audit:operational-loading
pnpm audit:domain
pnpm verify
pnpm build
```

## قاعدة البيانات

- قاعدة جديدة: شغّل `supabase/deploy.sql` مرة واحدة.
- قاعدة موجودة: شغّل `supabase/final-repair.sql` قبل نشر الكود الجديد.
- لا تشغّل الملفين على نفس القاعدة.

آخر تحديث لقاعدة البيانات يضيف:

- مجموعات وحدات صيدلية وقواعد دقيقة للكميات المعدودة والمقاسة.
- حماية قاعدة البيانات من الكسور في الحبة والقرص والحقنة والوحدات المعدودة.
- حالات تشغيل قانونية لأوامر الشراء والتوصيل والجرد.
- استلام جزئي أو كامل لأوامر الشراء داخل معاملة ذرّية واحدة.
- حماية Idempotency تمنع تكرار المخزون والحسابات عند إعادة إرسال طلب الاستلام.
- كشوف رواتب فعلية: مسودة، مراجعة، اعتماد، صرف، وقيد مالي ذرّي.
- حساب الراتب الشهري والأسبوعي واليومي والساعي باستخدام الحضور والإجازات وجدول العمل.
- احتساب نسبي للراتب الشهري عند التعيين أو التعطيل خلال منتصف الفترة.
- دورة موظف آمنة بالتعطيل وإعادة التفعيل بدل الحذف الذي يضيّع التاريخ.
- تحديد الحضور المتأخر من موعد الوردية مع فترة سماح افتراضية 15 دقيقة.
- State Machine للإجازات وقيود تمنع فترات الإجازة والساعات والرواتب غير المنطقية.

## قواعد الـ Domain

- القيم المالية تمر من `Money` باستخدام `decimal.js` ودقة منزلتين.
- الكميات تمر من `QuantityPolicy` حسب نوع الوحدة، وليس من تقريب عشوائي داخل الشاشات.
- الانتقالات التشغيلية تمر من State Machines في TypeScript ومن Triggers داخل PostgreSQL.
- عمليات المخزون تستخدم `ItemQuantityPolicyRepository` كحد موحد للتحقق.
- تحميل الصيدلية والفرع والصلاحيات والـ pagination يمر من `TenantRequestContext`.
- استدعاءات الواجهة تستخدم `src/lib/http/api-client.ts` كمصدر موحد.
- الموظفون والحضور والإجازات والرواتب تمر من Repositories وEnums وState Machines مشتركة.
- `AttendancePolicy` يوحد احتساب التأخير ويقرأ `hr.attendanceGraceMinutes` مع fallback آمن.
- صرف الرواتب يحتاج صلاحية HR وصلاحية مالية، ويسجل القيد والحركة مرة واحدة فقط.

## النشر

1. طبّق ملف SQL المناسب على Supabase.
2. اضبط متغيرات البيئة من `.env.example` داخل Vercel.
3. استخدم `pnpm install --frozen-lockfile` للتثبيت.
4. استخدم `pnpm run build` للبناء.
5. اختبر البيع والشراء والمرتجعات والجرد والتحويل والتوصيل بحسابات وصلاحيات مختلفة.
6. اختبر موظفًا شهريًا وآخر يوميًا/ساعيًا، ثم أنشئ كشف الرواتب واعتمده واصرفه.
7. تحقق من الحضور والإجازات المدفوعة وغير المدفوعة قبل اعتماد أول كشف فعلي.

لا يوجد زر منفصل لحساب المطور؛ يتم اكتشافه من مسار تسجيل الدخول العادي وفق إعدادات السيرفر.
